H2 Workshop - Enterprise Cross-Platform Infrastructure

  1. Home
  2. H2 Workshop Projekt

Projekt Information

  • Kategori: Enterprise Infrastructure
  • Skole: TEC (Teknisk Erhvervsskole Center)
  • Periode: Hovedforløb 2 (H2) - December 2022
  • Teknologier: Cisco IOS, Windows Server 2019, Rocky Linux 9.1, SQL Server, pfSense
  • Type: Multi-Platform Integration

Projekt Mål

Design og implementering af komplet enterprise infrastruktur med fokus på cross-platform integration, redundancy og security best practices på tværs af Cisco, Windows og Linux platforme.

Projekt Overblik

H2 Workshop projektet var et omfattende infrastructure projekt der demonstrerede enterprise-niveau kompetencer inden for netværksdesign, server administration og cross-platform integration. Projektet omfattede design og implementering af en komplet IT-infrastruktur med Cisco routing/switching, Windows Server services og Linux-baserede infrastructure services.

1. Netværk - Cisco L2/L3 Design

Kompleks enterprise netværksarkitektur implementeret med:

Routing & Switching Infrastructure:

  • Dual-Stack IPv4/IPv6: Komplet implementering af både IPv4 og IPv6 addressing på hele netværket
  • Dynamisk Routing: OSPF og EIGRP konfiguration til automatisk route propagation
  • VLAN Segmentation: Logisk netværkssegmentering baseret på organisationsstruktur med inter-VLAN routing
  • Spanning Tree Protocol: Rapid Spanning Tree (RSTP) implementering til loop prevention og redundancy
  • First Hop Redundancy: FHRP (HSRP/VRRP) til gateway redundancy og high availability

Security Implementation:

  • Access Control Lists (ACL): Granular traffic filtering og security policies
  • Switchport Security: Port security, BPDU Guard og PortFast konfiguration
  • Management VLAN: Dedikeret management VLAN til sikker administration

Infrastructure Services:

  • Quality of Service (QoS): Traffic prioritization og bandwidth management
  • NTP (Network Time Protocol): Synkroniseret tid på tværs af alle netværksenheder
  • Syslog: Centraliseret logging til alle Cisco devices
  • DHCP Integration: Windows Server DHCP relay via ip helper-address

2. Windows Server Infrastructure

Active Directory Domain Services:

  • Domain Controller: Windows Server 2019 AD DS implementering
  • DNS Integration: AD-integrated DNS zones med forward/reverse lookups
  • Group Policy Management: Centraliseret GPO deployment til workstations og servers
  • User/Computer Management: OU struktur og RBAC implementation

Infrastructure Roles:

  • DNS Services: Primary DNS server integreret med AD og Linux BIND
  • DHCP Server: IP address management på tværs af alle VLANs
  • Print Services: Centraliseret print server til organisationen
  • SQL Server: Database platform med sikker integration til applikationer

SQL Server Implementation:

  • Database Services: SQL Server installation og konfiguration
  • Security Hardening: SQL authentication, encryption og master keys
  • Cross-Platform Access: Database adgang fra både Windows og Linux clients
  • T-SQL Scripting: Stored procedures, functions og triggers til automation

3. Linux Infrastructure Services (Rocky Linux 9.1)

Firewall & Network Security:

  • Firewall Implementation: pfSense eller Linux-baseret firewall som perimeter security
  • NAT Configuration: PAT (Port Address Translation), SNAT/DNAT til internal/external traffic
  • Port Forwarding: Eksponering af interne services til eksterne netværk
  • Stateful Inspection: Connection tracking og advanced firewall rules

DNS Services (BIND 9):

  • Authoritative DNS: BIND version 9 implementering
  • IPv4/IPv6 Support: Dual-stack DNS records (A og AAAA records)
  • AD Integration: DNS forwarding og conditional forwarding til Windows DNS
  • Zone Management: Forward og reverse zones til LAN segmenter

Web Services:

  • Apache/Nginx: Webserver hosting til interne web applikationer
  • Virtual Hosts: Multiple websites på samme server
  • SSL/TLS: Sikker HTTPS kommunikation

Mail Server Infrastructure (Planlagt til Svendeprøve):

  • Postfix (SMTP): Mail Transfer Agent til sending/receiving af emails
  • Dovecot (IMAP): Mailbox access via IMAP protocol (port 143)
  • Roundcube: Webmail interface til browser-based email access
  • Configuration Files:
    • /etc/postfix/main.cf - Postfix hovedkonfiguration
    • /etc/dovecot/dovecot.conf - Dovecot IMAP konfiguration

System Services:

  • Syslog Server: Centraliseret log aggregation fra Cisco og andre Linux systemer
  • NTP Server: Time synchronization til hele infrastrukturen
  • SELinux: Security-Enhanced Linux policies for hardening
  • Firewalld: Dynamic firewall management med zones

Netværksadressering

  • ISP Connection: Fast IP-adresse range fra ISP (Skolens netværk)
  • Internal IP Schema: Private IP addressing med subnetting baseret på VLAN struktur
  • DMZ Network: Demilitarized Zone til firewall (Linux server) - Klasse C (192.168.x.x/24)
  • LAN Network: Klasse A netværk (10.10.x.x/16) til interne ressourcer

Cross-Platform Integration

Projektet demonstrerede seamless integration mellem platforme:

  • DNS Synchronization: Windows AD DNS og Linux BIND forwarders til unified name resolution
  • Time Synchronization: NTP hierarchy med Windows Server og Linux NTP servers
  • Logging Integration: Centraliseret Syslog modtager logs fra Cisco, Windows og Linux
  • Authentication: Potentiel Linux domain join til Windows AD (SSSD/Realmd)
  • Database Access: SQL Server tilgængelig fra både Windows og Linux clients

Kompetencer Demonstreret

  • Enterprise Network Design: Hierarkisk L2/L3 design med redundancy og high availability
  • Multi-Platform Administration: Cisco IOS, Windows Server og Linux i samme miljø
  • Security Best Practices: Defense in depth med ACLs, firewalls, segmentation og hardening
  • Service Integration: Seamless integration af DNS, NTP, Syslog på tværs af platforme
  • Documentation: Komplet teknisk dokumentation af design og implementering
  • Troubleshooting: Systematisk fejlsøgning på tværs af OSI-lag og platforme

Projektresultat

H2 Workshop projektet demonstrerede mine kompetencer inden for design og implementering af komplekse, multi-platform enterprise infrastrukturer. Projektet viste evnen til at integrere forskellige teknologier (Cisco, Windows, Linux) i et sammenhængende, sikkert og effektivt infrastruktur setup.

Dette projekt var fundamentet for min videre specialisering i infrastructure automation og dannede basis for senere projekter med Proxmox virtualisering, Ansible automation og advanced Cisco networking.
