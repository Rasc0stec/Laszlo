Cisco Netværk – 4 års praktisk arbejde

Siden 2022 har jeg arbejdet intensivt med Cisco-netværksteknologier på tværs af hjemmelab, studieprojekter og enterprise-designprojekter. Alle konfigurationer herunder er egne, praktiske implementeringer — ikke teoretiske gennemgange. Materialet dækker 148 filer, 30+ device-konfigurationer og 66+ ENARSI labs over 4 år.

EIGRP for IPv6 — CCNP Lab

Named EIGRP vs Classic EIGRP — Wide Metrics & Unequal Load Balancing CCNP

EIGRPv6 Named EIGRP Wide Metrics (64-bit) Variance / Unequal LB HMAC-SHA-256 Prefix-list filtering
  • R1 og R3: Named EIGRP (router eigrp EIGRP_IPV6, AS 43) — R2: Classic EIGRP — begge i samme AS, forklaret metrisk inkompatibilitet
  • Wide metrics (64-bit) analyseret: RIB-scale factor 128 giver forskel mellem topology table og routing table (FD 2048000 → RIB 16000)
  • Unequal-cost load balancing med variance 2 — feasibility condition verificeret manuelt inden aktivering
  • Summary routes på interface-niveau: ::/0 (default) og 2001:db8:abcd::/56 (5 loopbacks aggregeret)
  • Route filtering: IPv6 prefix-list DROP-CAFE-1 med distribute-list ud på specifikt interface mod R3
  • Dual authentication: MD5 key-chain (Classic) og HMAC-SHA-256 (Named) — ikke kompatible, konfigureret separat per interface
! Named EIGRP — R1: router eigrp EIGRP_IPV6 address-family ipv6 unicast autonomous-system 43 af-interface g0/0/0 authentication key-chain EIGRPv6-AUTHEN-KEY authentication mode md5 af-interface g0/0/1.1 authentication mode hmac-sha-256 $3cre7!! topology base variance 2
BGP — Advanced Routing

MP-BGP, Path Manipulation og Route Redistribution CCNP

eBGP iBGP MP-BGP AS-PATH / MED / Local-Pref BGP Communities Redistribution EIGRP↔BGP
  • eBGP peering konfigureret på tværs af multiple AS'er — path manipulation via AS-PATH prepending, MED og Local Preference

DMVPN — Dynamic Multipoint VPN

DMVPN Phase 1 & Phase 3 med IPv6 Dual-Stack CCNP

DMVPN Phase 1 DMVPN Phase 3 NHRP GRE Multipoint IPv6 Dual-Stack NHRP Auth
  • Hub konfigureret med tunnel mode gre multipoint, NHRP network-id, ip nhrp map multicast dynamic og ip nhrp redirect
  • Phase 3 Spokes: ip nhrp shortcut for direkte spoke-to-spoke kommunikation uden hub-forwarding
  • Dual-stack: IPv4 og IPv6 tunnel-adresser (2A02:188:5204:FAC0::/128 og FAC1::/128) på separate tunnel interfaces
  • Multiple tunnel interfaces med tunnel key for at skelne trafikken på samme tunnel source
  • NHRP authentication konfigureret, NHRP registration no-unique for DHCP-baserede transport-interfaces
  • Troubleshooting: recursive routing identificeret og løst ved at stoppe routing-annoncering af tunnel source-netværk

IPsec & GRE

GRE over IPsec Site-to-Site VPN CCNP

IPsec IKEv1/v2 ISAKMP / ESP GRE Tunnel AES-256 / SHA-256 DH Group 14+
  • IPsec site-to-site VPN implementeret med IKEv1/v2 — ISAKMP policy, transform-set og crypto map konfigureret
  • GRE tunnel over IPsec for at transportere OSPF og multicast-trafik igennem den krypterede tunnel
  • Kryptering: AES-256 med SHA-256 og DH Group 14 (minimum) for at undgå svage cipher-suites
  • Forstår distinction: GRE alene er ikke sikret — IPsec tilføjer fortrolighed, integritet og autentificering
  • Dokumenteret i eget lab-dokument med topologi, konfigurationstrin og verifikationskommandoer

Hjemmelab — Cisco 3750X & 3850

Multi-VLAN Hjemmenetværk med L3-Switching, ACL-Isolation og QoS

L3 Switching VLAN Segmentering Extended ACLs DHCP Server QoS DSCP PortFast / STP
  • 5 VLANs: Mgmt (10), Trusted (20), IoT (30), Guest (40), WAN (99) — alle med SVI og DHCP-pool
  • GUEST_ISOLATION ACL: blokerer al adgang til RFC1918-adresser, tillader kun internet
  • IOT_INTERNET_RULES ACL: IoT-enheder isoleret fra interne netværk men har internetadgang
  • MGMT_PROTECTION ACL: kun Trusted-netværket (VLAN 20) kan tilgå management-SVI
  • QoS policy med DSCP EF (voice priority 15%) og DSCP AF41 (video remaining 50%) på WAN-interface
  • Trunk-porte til access points med native VLAN 10 og allowed VLANs per SSID
  • Konfigureret på både 3750X (IOS) og 3850 (IOS-XE) — platform-forskelle dokumenteret
