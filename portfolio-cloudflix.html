CloudFlix A/S (H5)

  1. Home
  2. Project Details

Projektoversigt & Formål

For H5-projektet "CloudFlix A/S" var opgaven at designe og implementere kernen af en ny IT-infrastruktur for en ny afdeling i Aarhus. Hovedfokus var at skabe en skalerbar og vedligeholdelsesvenlig løsning, hvor manuelle processer blev elimineret gennem omfattende automatisering med PowerShell. Dette sikrede en hurtig, ensartet og fejlfri opsætning af centrale server-roller.

Projektinformation

  • Kategori: Infrastruktur & Automatisering
  • Teknologier: PowerShell, Active Directory, DFS, Windows Server
  • Projekt Dato: August - December, 2025

Kernekompetencer i Projektet

  • PowerShell Scripting: Udvikling af idempotente scripts til fuldautomatiseret server-konfiguration.
  • Active Directory: Design og automatiseret oprettelse af OU-struktur, grupper og brugere.
  • Distributed File System (DFS): Automatiseret opsætning af DFS Namespace og Replikering for redundant fildeling.
  • Windows Server Administration: Dybdegående konfiguration af server-roller og -features.

Implementering via PowerShell

Hele server-opsætningen blev kodificeret i to primære PowerShell-scripts:

1. Automatiseret Active Directory Setup (`AD_Setup_Aarhus.ps1`)

Dette script læser en foruddefineret struktur og opretter automatisk alle OU'er, sikkerhedsgrupper og brugere. Dette sikrer en hurtig og konsistent opsætning og minimerer risikoen for manuelle fejl.

# Uddrag fra AD_Setup_Aarhus.ps1
function New-AarhusUser {
    param(
        [string]$FullName,
        [string]$SamAccountName,
        # ... andre parametre
    )
    # ... logik til at oprette bruger i korrekt OU
    New-ADUser @userParams
    # ... logik til at tilføje bruger til grupper
}
2. Automatiseret Filserver & DFS Setup (`FileServer_Setup_Aarhus.ps1`)

Et idempotent script der klargør to filservere til redundant drift. Scriptet håndterer installation af Windows Features (DFS), oprettelse af mappestruktur, tildeling af komplekse NTFS-rettigheder, opsætning af SMB-shares og fuld konfiguration af DFS Namespace og replikering.

# Uddrag fra FileServer_Setup_Aarhus.ps1
# Opretter DFS Namespace og tilføjer begge servere som targets
New-DfsnRoot -Path $namespace -Type DomainV2 -EnableAccessBasedEnumeration $true
New-DfsnRootTarget -Path $namespace -TargetPath "\\$fileServer01\Shares"
New-DfsnRootTarget -Path $namespace -TargetPath "\\$fileServer02\Shares"
